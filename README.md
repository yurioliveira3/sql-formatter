# sql-formatter

SQL formatter for ANSI/generic SQL, designed as an external formatter for DBeaver. Reads from stdin, writes to stdout.

## How it works

The pipeline applies formatting in order:

1. **preserve block comments** — original `/* */` in the input are replaced with placeholders so sqlglot doesn't touch them
2. **sqlglot** — parses and pretty-prints the SQL (handles SELECT, INSERT, UPDATE, CTE, subqueries, etc.)
3. **FILTER merge** — reassembles `FILTER(WHERE ...)` clauses that sqlglot splits across lines
4. **expand block comments** — `/* */` that came from `--` are expanded back into individual `--` lines (3 patterns: pure-comment lines, AND/OR with leading comments, code with trailing comments)
5. **convert block comments** — remaining single-line `/* */` are converted to `--`
6. **restore block comments** — original `/* */` placeholders are restored to their original form
7. **FROM / JOIN layout** — moves table names to the next line with a tab indent; `ON (...)` stays on the same line as the table
8. **LIMIT layout** — moves the limit value to the next line with a tab indent
9. **MERGE layout** — custom handler for MERGE statements (sqlglot has no reliable ANSI MERGE support)
10. **finalize** — normalizes CRLF, strips trailing semicolons, adds a single `;\n` at the end

### Output style example

```sql
SELECT
  COUNT(DISTINCT o.id) FILTER(WHERE li.id IS NULL) AS orders_sem_line_items,
  name
FROM
	users
INNER JOIN
	orders ON users.id = orders.user_id
LEFT JOIN
	items ON items.order_id = orders.id
WHERE
  active = 1
LIMIT
	10
;
```

## Files

| File | Description |
|---|---|
| `format_sql_ansi.py` | Main formatter script |
| `format_sql_ansi.bat` | Entry point called by DBeaver |
| `test_format_sql.py` | pytest suite (unit + integration) |
| `dist/format_sql_ansi/` | Standalone executable generated by PyInstaller |

## Requirements

### To run the Python script directly
- Python 3.10+
- `sqlglot`: `py -3 -m pip install sqlglot`

### To rebuild the executable
- PyInstaller: `py -3 -m pip install pyinstaller`

## Running

### Via .bat (DBeaver integration)
The `.bat` calls the standalone exe — no Python required on the target machine:
```bat
"C:\projects\sql-formatter\dist\format_sql_ansi\format_sql_ansi.exe"
```

### Via Python directly
```bat
py -3 format_sql_ansi.py
```

### Manual test (stdin)
```bat
echo SELECT id FROM users WHERE active = 1 | dist\format_sql_ansi\format_sql_ansi.exe
```

## Building the executable

```bat
cd C:\projects\sql-formatter
py -3 -m PyInstaller --onedir --name format_sql_ansi --console -y format_sql_ansi.py
```

Output: `dist\format_sql_ansi\format_sql_ansi.exe`

Use `--onedir` (not `--onefile`) to avoid the per-execution extraction overhead that `--onefile` incurs.

## Running tests

```bat
py -3 -m pytest test_format_sql.py -v
```

Tests call the script via subprocess, so they exercise the full pipeline end-to-end.

## DBeaver configuration

1. Open **Window → Preferences → Editors → External Formatters**
2. Add a new formatter pointing to:
   ```
   C:\projects\sql-formatter\format_sql_ansi.bat
   ```
3. Set the formatter for SQL dialect and save.

## Limitations

- MERGE formatting is handled by a custom regex-based layout, not sqlglot. Complex or non-standard MERGE syntax may not format perfectly.
- Multi-statement scripts (more than one statement separated by `;`) are not the primary use case.
- `--` comments that appear before SELECT columns are moved by sqlglot to after the last real column. The original order is not recoverable without removing comments before passing to sqlglot.
